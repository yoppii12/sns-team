# sns-team

SNS運用自動化 運営チームのプロジェクトリポジトリ。X（Twitter）・note・YouTubeでの発信を、Claude Codeの複数エージェント体制で自動化する。

## Cowork と Claude Code の使い分け

- **Cowork（日常運用）**：ネタ収集→企画→下書き→確認→予約→振り返りの運用ループは、Notion／Slack／Googleドライブの接続が認証済みのCoworkで行う。このリポジトリの `.claude/agents/` 7ファイルと `CLAUDE.md` は、Cowork上では「常駐エージェント」ではなく、Claudeが読み込んで従う役割定義として機能する
- **Claude Code（このリポジトリ・技術拡張）**：X・noteへの実投稿の実装など、公式APIが無い、または開発作業を伴う拡張はこちらで行う

## 対象SNSと運用方針

- 対象媒体：X（Twitter）、note、YouTube
- 当面は1媒体1アカウントに統合（目的別・媒体別の分割はしない）。noteはマガジン機能、YouTubeは再生リストで目的別に整理
- 拡張候補：Threads（X運用が軌道に乗ってから）→ Instagram（ビジュアル制作体制が整ってから）
- **X**：公式API（X API v2、従量課金制）で自動化する方針。X Developerアカウント作成・APIキー発行が前提
- **note**：公式APIが無いため要検証（非公式API/ブラウザ自動化はリスクを伴う）。当面は「Notion下書き→人が手動でコピペ公開」

## 3つの発信目的

すべての投稿に目的タグを付与する。

- `#lead` セールスリード獲得
- `#pr` 自社PR
- `#recruit` 採用

## チーム構成（7エージェント）

`.claude/agents/` 配下に定義。工程（役割）軸で分割し、媒体・目的はエージェントへの入力パラメータとして渡す設計。

| エージェント | 役割 | 権限 |
|---|---|---|
| content-strategist | 目的ポートフォリオ管理・ネタ選定 | 提案まで |
| source-miner | Slack議事録・Googleドライブの文字起こしからネタ抽出 | 自動 |
| copywriter | 媒体別下書き執筆 | 下書きまで |
| creative-producer | 画像・サムネ・動画編集案 | 提案まで（動画編集の完全自動化は要検証） |
| scheduler | 投稿カレンダー・予約投稿 | 承認済みのみ自動実行 |
| engagement | コメント・DM返信下書き | 下書きまで、送信は人 |
| reporter | 効果測定・比率の可視化 | 自動 |

公開判断、DM・コメントの送信、誹謗中傷への対応は必ず人が行う。エージェントに最終判断はさせない。

## データソース（接続検証済み）

- **Slack**：`#general`（月火水のハドル、議事録機能で要約済み）、`#general-dev`（エンジニア定例）
- **Googleドライブ**：「LAplust Scrum」フォルダ内のGoogle Meet議事録（Gemini自動生成）
- **Notion**：「SNS Team」ページ内の「下書き置き場」データベースがネタ・下書きの管理場所。プロパティはステータス（下書き中／要チェック／承認済み／公開済み）、媒体、目的タグ、担当エージェント、機密チェックなど

## 運用フロー・頻度

```
source-miner（ネタ抽出）
  → content-strategist（ネタ選定・比率管理）
  → copywriter（媒体別下書き作成）
  → 人による秘密情報チェック・仕上げ
  → scheduler（承認済みのみ予約投稿）
```

| 頻度 | 内容 | 人の作業 |
|---|---|---|
| 毎日（自動） | source-miner がSlack/Googleドライブの新規メモを確認しNotionへネタ登録、content-strategist が優先順位を提案 | ―（自動実行） |
| 毎日（人） | Notionのネタ候補・下書きを確認 | 約10分：承認/修正/却下、要チェック項目の確認、DM送信 |
| 週次 | content-strategist が投稿計画（目的タグ配分）を提示 | 約15分：優先度調整、schedulerへ予約指示 |
| 月次 | reporter がKPI比較レポートを作成 | 約30分：比率・目標の見直し |

## セキュリティ運用ルール

- パスワード等の認証情報はチャットに直接貼らない。OAuth連携が可能なものは連携方式に寄せる
- 誤って共有してしまった認証情報は、共有された時点で漏洩したものとして速やかにローテーションする
