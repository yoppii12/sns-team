# SNS運用自動化 運営チーム セットアップ手順

対象読者：Claude Codeで「SNS運営チーム」を動かしたいyoshikaさん向け。設計の背景・全体像は別紙「260711_SNS運用自動化チーム設計書.docx」を参照。

## 1. ファイルの配置

同梱の7つのエージェント定義ファイル（`260711_agent_*.md`）を、Claude Codeのプロジェクトフォルダ内の `.claude/agents/` に配置してください。

```
your-project/
└── .claude/
    └── agents/
        ├── content-strategist.md   ← 260711_agent_content-strategist.md を配置
        ├── source-miner.md
        ├── copywriter.md
        ├── creative-producer.md
        ├── scheduler.md
        ├── engagement.md
        └── reporter.md
```

ファイル名の日付プレフィックス（`260711_agent_`）は納品物としての命名規則によるものです。`.claude/agents/` に置く際はファイル名を自由に変更して構いません（Claude Codeはファイル内のfrontmatterの `name` でエージェントを識別します）。

## 2. 必要な外部連携（MCP）

各エージェントが実際に外部サービスを操作するには、Claude CodeでMCPサーバーの接続設定が必要です。

| 連携先 | 使うエージェント | 用途 |
|---|---|---|
| Googleドキュメント | source-miner | 社内文字起こしの検索・読み込み |
| Notion | source-miner, copywriter, scheduler | 下書きの登録・更新 |
| Slack（必要であれば） | source-miner | Slack上の議事録・日報の参照 |

設定方法は Claude Code のドキュメント（MCP連携）を参照し、`.mcp.json` にそれぞれのサーバーを登録してください。各エージェント定義ファイルの `tools:` 欄に書かれているMCPツール名（例：`mcp__notion__notion-search`）は、実際に接続したMCPサーバー名に合わせて調整が必要な場合があります。

X（Twitter）・note・YouTubeへの実際の投稿・予約投稿については、現時点では専用のMCP連携先が未確定です。連携方法（各サービスの公式API、またはBufferなどの外部予約投稿ツール経由）は別途検討が必要です。

## 3. 基本的な運用の流れ

1. **ネタ収集**：`source-miner` を実行し、直近の文字起こしからネタ候補をNotionに集める
2. **企画**：`content-strategist` に「今回は採用向けのネタが欲しい」のように目的を伝えて選定してもらう
3. **執筆**：`copywriter` に下書きを作成させる（媒体別フォーマットに変換）
4. **確認**：yoshikaさんが下書きを確認。「要チェック」フラグの箇所は必ず目視確認する
5. **予約**：承認後、`scheduler` に予約投稿を依頼する
6. **反応対応**：コメント・DMが来たら `engagement` に下書きを作らせ、内容を見てから送信する
7. **振り返り**：週次・月次で `reporter` にレポートを作成させ、次の企画（1に戻る）に活かす

## 4. 運用開始前に決めておくこと

- Notion上の「下書き置き場」のデータベース構成（ステータス列に「要チェック」「承認済み」等を用意しておくと scheduler が判定しやすい）
- 秘密情報の最終チェックは必ずyoshikaさんが行う（エージェントはフラグを立てるだけで、削除・公開判断はしない設計）
- YouTubeの動画編集自動化は「要検証」の段階。まずは `creative-producer` にカット・テロップの提案をさせるところから試し、実用に耐えるか確認してから本格導入する

## 5. 今後の拡張

- Threads追加時：`copywriter` の媒体別フォーマット方針にThreads向けの項目を追記するだけで対応可能
- Instagram追加時：`creative-producer` の担当範囲が増えるため、先にビジュアル制作の体制を整えてから着手
- 採用の比重が大きくなった場合：Xアカウントの分割（採用垢の切り出し）を検討し、`content-strategist` の目的タグ管理に「媒体」も紐付ける形に拡張する
