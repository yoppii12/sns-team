# SNS運用自動化 運営チーム — プロジェクト前提

このファイルはClaude Codeが起動時に自動で読み込む前提情報です。各エージェント（`.claude/agents/`以下）はこの前提を共有して動くこと。

## 中の人・発信スタイル
- 発信の「中の人」はyoshikaさん本人。実名でのコミュニケーションを基本とする
- 社員個人の発言も実名表記でよい（匿名化しない）

## 対象SNSと現状の運用方針
- 対象媒体：X（Twitter）、note、YouTube
- アカウントは目的別・媒体別に分割せず、当面は1媒体1アカウントに統合する
- noteは無料のマガジン機能、YouTubeは再生リストで目的別に整理する
- 拡張候補：Threads（X運用が軌道に乗ってから）→ Instagram（ビジュアル制作体制が整ってから）
- note・YouTubeへの投稿実行に公式APIは無い。noteは非公式API/ブラウザ自動化のみで不安定なため、現状は「Notion下書き→人が手動でコピペ公開」が基本方針。YouTubeは本人出演・撮影

## 3つの目的（すべての投稿に目的タグを付与）
- `#lead` セールスリード獲得
- `#pr` 自社PR
- `#recruit` 採用
- 投稿比率はyoshikaさんが都度明示する。指定が無い場合はreporterエージェントの直近実績を見て偏りを是正する方向で提案する

## KPI（仮置き・実績を踏まえて要更新）
| 指標 | 立ち上げ〜3ヶ月 | 4〜6ヶ月 |
|---|---|---|
| Xフォロワー数 | 300〜500 | 800〜1,500 |
| YouTube登録者数 | 50〜100 | 150〜300 |
| 問い合わせ数（リード） | 月1〜3件 | 月3〜8件 |
| SNS経由サイト流入 | 月50〜100 | 月150〜300 |
| 採用カジュアル面談応募 | 月0〜1件 | 月1〜2件 |

## 秘密情報チェック基準（自動ブロックはしない。「要チェック」として明示し、公開判断は必ずyoshikaさんが行う）
- 顧客名・取引先名
- 契約情報（金額・条件）
- 未公開の数値実績
- 開発中の未発表機能
- 社内の人間関係に関するネガティブな発言

上記のうち、下書き本文への含め方はcopywriterの段階で次のように使い分ける（`copywriter.md`参照）。
- **顧客名・取引先名／契約情報／社内の人間関係に関するネガティブな発言**：本文に含めない。含める必要がありそうな場合は「【要相談】」として具体的に何を判断してほしいか記載する
- **未公開の数値実績／開発中の未発表機能**：機密性が高いと判断した場合でも、具体的な数値・詳細を含めて下書きを作成する。「【要相談】」として、公開してよいか・伏せるべきかyoshikaさんに判断を仰ぐ

## エンゲージメント対応方針
- DM返信：下書きまでエージェントが作成。送信は必ずyoshikaさんが行う
- ネガティブコメント：基本テンプレート「コメントいただきありがとうございます」で御礼を伝える
- 誹謗中傷・深刻なクレーム：自動返信せず、yoshikaさんにエスカレーションする

## フィードバックによる改善ループ
Notion「下書き置き場」データベースに「フィードバック」欄（テキスト）を用意している。yoshikaさんが下書きを確認する際、指摘事項をこの欄に残す。

```
copywriterが下書きを作成
  → yoshikaさんが下書きを確認し、「フィードバック」欄に指摘を残す
  → copywriterは次回以降、同じ媒体・目的タグの過去のフィードバックを参照して執筆する（即時反映）
  → reporterが月次レポート作成時、繰り返し出てくる指摘傾向をまとめ、エージェント定義（`.claude/agents/*.md`）への反映を提案する
  → yoshikaさんが提案を確認し、承認したものだけを実際にエージェント定義に反映する
```

エージェント定義ファイルの書き換えは自動では行わない。フィードバックの傾向抽出・提案までが自動、実際の定義変更は必ず人が承認する。

## 社内一次情報の活用フロー
Googleドキュメントに保存された社内ミーティング・日報の文字起こしを、生きた一次情報として活用する。

`source-miner`（Googleドキュメントを検索・ネタ抽出・目的タグ付け・Notionへ登録）→ `content-strategist`（ネタ選定・比率管理）→ `copywriter`（媒体別下書き作成）→ yoshikaさんが秘密情報チェック・仕上げ → `scheduler`（承認済みのみ予約投稿）

## チーム構成（7エージェント、`.claude/agents/`参照）
| エージェント | 役割 | 権限 |
|---|---|---|
| content-strategist | 目的ポートフォリオ管理・ネタ選定 | 提案まで |
| source-miner | 社内一次情報からネタ抽出 | 自動 |
| copywriter | 媒体別下書き執筆 | 下書きまで |
| creative-producer | 画像・サムネ・動画編集案 | 提案まで（動画編集の完全自動化は要検証） |
| scheduler | 投稿カレンダー・予約投稿 | 承認済みのみ自動実行 |
| engagement | コメント・DM返信下書き | 下書きまで、送信は人 |
| reporter | 効果測定・比率の可視化 | 自動 |

## 使い方の原則
- 各エージェントは明示的に名指しして依頼するのが確実（例：「content-strategistエージェントを使って〜」）
- 秘密情報チェック後の公開判断、DM・コメントの送信、誹謗中傷への対応は必ずyoshikaさんが行う。エージェントに最終判断をさせない
- 詳細な背景・意思決定の経緯は設計書（`260711_SNS運用自動化チーム設計書.docx`）を参照
